# OverlayMusic v1.1.0 — Прозрачный режим

Добавлен полностью прозрачный режим оверлея + мелкие фиксы.

## Что нового

- ✨ **Полностью прозрачный режим** (трей → Настройки → «Внешний вид») — фон, подложка и тень не рисуются, на экране остаются только обложка, текст и кнопки. Удобно поверх тёмных игр; на светлом фоне текст может быть хуже читаем — выключай если мешает.
- 🐛 Исправлено сохранение настроек: при нажатии «Сохранить» в диалоге настроек могла молча падать валидация хоткея, и новые значения не применялись.

## Установка

### 🪟 Windows

| Файл | Что это |
|---|---|
| **`OverlayMusic-Setup-1.1.0.exe`** | Установщик (рекомендуется). Требует прав администратора. Ставит в Program Files и активирует автозапуск через Scheduled Task — оверлей будет запускаться при каждом входе в Windows автоматически и **без UAC-промпта**. |
| **`OverlayMusic.exe`** | Портативный exe — запусти и работает. Хочешь работу поверх игр от админа (CS, Valorant) — запускай ПКМ → «Запуск от имени администратора». |

#### ⚠ SmartScreen / Defender

Файлы не подписаны Authenticode-сертификатом (~€300/год — для опенсорса избыточно). Windows покажет **«Защитник Windows предотвратил запуск...»** — это штатно для любого нового неподписанного приложения.

**Что делать:** в окне предупреждения жми **«Подробнее» → «Выполнить в любом случае»**. Или заранее: ПКМ по .exe → Свойства → внизу галочка **«Разблокировать»**.

### 🍏 macOS (Apple Silicon — M1, M2, M3)

| Файл | Что это |
|---|---|
| **`OverlayMusic-arm64.dmg`** | Образ для перетаскивания в Applications. Внутри `media-control` CLI (ungive) для извлечения now-playing данных — работает на macOS 15.4+ где Apple зарубила прямой доступ к MediaRemote. |

**Установка:**
1. Открой `.dmg`, перетащи **OverlayMusic** в **Applications**
2. Первый запуск: **ПКМ по приложению → Open** → подтверди (приложение не подписано Apple Developer ID)
3. Если совсем не запускается: `xattr -dr com.apple.quarantine /Applications/OverlayMusic.app`

**Хоткей по умолчанию на Mac:** `Cmd+Opt+]` (можно поменять в Настройках)

Глобальный хоткей реализован через **Carbon RegisterEventHotKey** — не требует Accessibility-разрешения, работает поверх любых полноэкранных игр.

## Использование

| Действие | Hotkey (Win) | Hotkey (Mac) |
|---|---|---|
| Показать / скрыть оверлей | `Ctrl+Alt+S` | `Cmd+Opt+]` |
| Предыдущий трек (в фокусе оверлея) | `←` | `←` |
| Следующий трек | `→` | `→` |
| Пауза / воспроизведение | `↓` | `↓` |
| Скрыть оверлей | `Esc` | `Esc` |

Хоткеи настраиваются: трей → правый клик → **Настройки**.

## Что работает

- ✅ Любой плеер с поддержкой системных media-controls — **Spotify, Yandex.Music, AIMP, foobar2000, Tidal, MusicBee, Apple Music, любой браузер с YouTube/SoundCloud**
- ✅ **Cover-derived theming** — обложка размывается в фон, прогресс-бар и play-кнопка красятся под акцент
- ✅ **Полностью прозрачный режим** — для минималистов и тех, кто хочет видеть только инфу о треке
- ✅ **Discord Rich Presence** с обложкой, временем, кнопкой «Открыть трек» в Я.Музыке (или Spotify / YouTube / Apple — на выбор)
- ✅ Прямые ссылки на трек в Яндекс.Музыке через их публичный API
- ✅ **Запуск с правами админа** на Windows → работает поверх игр-античитов
- ✅ **Автозапуск без UAC-промпта** через Scheduled Task

## Известные ограничения

- **Яндекс.Музыка не репортит позицию в SMTC** — оверлей считает время локально от смены трека. Первый трек после запуска может показывать неточную позицию; со второго всё точно.
- В Discord **кнопка «Открыть трек» не показывается на твоём собственном профиле** — это политика Discord, она видна другим пользователям.
- На macOS 15.4+ `media-control` CLI работает только если **Now Playing** активен (плеер запущен и трек загружен) — в первые 1-2 секунды после запуска плеера может показывать «Музыка не играет».
